🖼️ 头像备份恢复工具

解决拷贝到新环境后头像丢失的问题

💡 为什么头像会丢失？
头像以 base64 格式存在浏览器的 IndexedDB（本地数据库）中，这个数据库不随代码文件迁移。
换电脑、换浏览器、清除缓存、部署到新服务器——都会导致头像消失。

解决方法：用此工具在旧环境导出头像，再在新环境导入恢复。

📤 在旧环境：导出头像

1
在有头像的旧环境（浏览器）打开此工具
确保上方状态面板显示"有 X 个学生有头像"
2
点击下方按钮导出头像文件
会下载一个 avatars-backup-YYYYMMDD.json 文件，请妥善保存

📥 在新环境：导入恢复头像

1
在新环境（目标浏览器）打开此工具
确保系统数据已正常导入（学生列表不为空）
2
选择从旧环境导出的头像备份文件
选择 avatars-backup-*.json 文件，工具会自动将头像写回对应学生

💡 其他恢复方式

如果无法访问旧环境，还有以下备选方案：
1. 系统内导入 JSON 全量备份：管理员登录 → 设置 → 数据管理 → 导入 JSON
2. 云端同步：管理员登录 → ☁️ 云端同步 → 点击"📥 恢复头像"（需先在旧环境备份过）
3. 让学生重新上传：登录学生账号 → 宠物页面 → 点击📷 重新上传头像
