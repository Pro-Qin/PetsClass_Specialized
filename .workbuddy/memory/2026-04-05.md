### 购物后错误弹窗修复（2026-04-05 13:34）

| 文件 | 修复内容 |
|------|---------|
| **student.js (StudentApp data)** | 新增 `_isPurchasing: false` 购物中标志 |
| **student.js (StudentApp template)** | PetPage 监听 `@purchase-start` / `@purchase-end`，切换 `_isPurchasing` |
| **student.js (PetPage buyItem)** | 买前 emit `purchase-start`，买后 emit `purchase-end` |
| **student.js (startPointsWatch)** | `isBuyDeduct` 条件加入 `this._isPurchasing` 判断，购物期间积分变化不弹窗 |

**根本原因**：`Store.buyItem` 异步写入 IndexedDB 后，`refreshStudent()` 从 IndexedDB 重新读，此时 `_buyDeduct` 标记已丢失，定时器检测到扣分却误判为"老师扣分"弹出通知。

### 购物弹窗二次修复 + 每日成长逻辑（2026-04-05 13:39）

**问题**：购买后仍显示老师扣除弹窗，原因是 `_isPurchasing` 在 `refreshStudent` 后被 IndexedDB 数据覆盖，且 `_buyDeduct` 时序不稳定。

**新策略**：购买前直接快照积分，检测积分变化时对比差值是否等于购买消耗，完全绕过 DB 持久化标志。

| 文件 | 修复内容 |
|------|---------|
| **student.js (StudentApp data)** | 新增 `_lastBuyPoints: null`（购买前快照积分）、`_lastBuyCost: 0`（本次购买消耗） |
| **student.js (PetPage emits)** | 声明 `purchase-start`/`purchase-end` 事件 |
| **student.js (PetPage buyItem)** | emit 时传 `{ points, cost }` 快照 |
| **student.js (StudentApp template)** | `@purchase-start="onPurchaseStart"` `@purchase-end="onPurchaseEnd"` |
| **student.js (onPurchaseStart/End)** | 快照 + 标志管理 |
| **student.js (startPointsWatch)** | `isBuyDeduct` 改为 `this._lastBuyPoints !== null && delta === -this._lastBuyCost` |
| **data.js** | `DAILY_EXP_LIMIT` 50 → 200（每日经验上限） |
| **api/students.php** | 同步改为 200 |
| **student.js (runDailyPenaltyCheck)** | 新增每日成长逻辑：按日期差给 50 积分，上限 200，用 `lastGrowth_${id}` localStorage 键控制每日一次 |
